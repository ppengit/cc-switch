# 修复复核与上游同步记录

## 1. 背景

本轮整改基于用户安装导入后的实际反馈，目标不是只修 UI，而是复核上游最新版本和本地定制之间的冲突点，保证代理、接管、故障转移、配置模板、请求日志、用量统计等核心流程不再被破坏。

复核基准：

- 上游仓库：`https://github.com/farion1231/cc-switch`
- 本地目标仓库：`https://github.com/ppengit/cc-switch`
- 已同步上游提交截止：`eb304232`

## 3. 用户反馈问题整改

### 3.1 配置模板多文件化

- 配置模板由单个文本改为 `AppConfigTemplateFile[]`。
- Gemini 默认拆为 `.env` 与 `settings.json` 两个文本域。
- 当前配置弹窗直接展开全部实际配置文件，不再使用下拉选择。
- 配置模板弹窗增加“加载环境配置”。
- 模板保存增加“保存后更新实际配置”，保存时可立即同步 live 文件。

### 3.2 旧通用配置退出主流程

- 添加/编辑供应商不再展示旧“通用配置”编辑入口。
- 新增 `meta.useConfigTemplate`，默认启用。
- `useConfigTemplate=false` 时，live 写入直接使用供应商原始配置。
- 保存供应商不再剥离公共片段，数据库供应商配置保持单一事实源。

### 3.3 Gemini API Key 丢失

根因：

- 编辑时如果读取 live 配置，拿到的是模板渲染后的环境文件内容，可能与数据库供应商原始配置不一致。
- 表单中 API Key 输入依赖 React state，极端情况下立即保存可能读到旧状态。

修复：

- 编辑供应商只读取数据库 provider，不再用 live 配置覆盖表单。
- Gemini API Key/Base URL/Model 输入会同步回隐藏的 `settingsConfig` JSON。
- 提交 Gemini 时再次以输入框 state 覆盖 `settingsConfig.env` 中的 `GEMINI_API_KEY`、`GOOGLE_GEMINI_BASE_URL`、`GEMINI_MODEL`，避免最后一次输入未刷新导致保存丢失。

### 3.4 置顶后序号重复

- 置顶按钮图标改为 `ArrowUpToLine`。
- 排序保存后端新增归一化：同应用所有供应商会重新写成连续 `sort_index=0..n`。
- 表格序号继续按展示顺序或故障转移队列顺序计算，不直接显示数据库原始 `sort_index`。
- 添加供应商默认勾选置顶；编辑供应商默认不置顶，避免无意改变故障转移顺序。

### 3.5 熔断、降级、故障转移状态恢复

- 供应商列表状态列重新接入 `useProviderHealth` 与 `useCircuitBreakerStats`。
- 状态列显示：
  - 当前代理
  - 已启用/已禁用
  - 熔断
  - 半开
  - 降级及连续失败次数
- `get_circuit_breaker_stats` 改为从运行中的 `ProxyServer -> ProviderRouter` 读取真实统计，不再返回空值。

### 3.5.1 代理接管期间模板保存/热切换安全修复

复核发现配置模板保存会直接触发 live 同步，如果应用正处于代理接管状态，直接写 live 会把真实供应商地址和真实 Key 写回应用配置，导致绕开代理、故障转移失效。

修复后：

- `set_app_config_template` 不再直接调用底层 live 写入，而是走 `ProviderService::sync_current_provider_for_app`。
- 当前供应商更新、模板保存、代理热切换均使用同一条接管安全路径：
  1. 通过模板生成有效供应商配置。
  2. 更新 live backup，确保停止接管时恢复到最新配置。
  3. 写接管态 live，覆盖为本地代理地址和 `PROXY_MANAGED` 占位 Key。
- 已覆盖 Claude / Codex / Gemini 三个支持接管的应用。
- 新增回归测试：
  - `proxy_active_sync_renders_codex_template_without_leaking_real_credentials`
  - `proxy_active_sync_renders_gemini_template_without_leaking_real_credentials`

### 3.6 请求日志详情

- 请求日志列表显示会话标题和项目路径。
- 双击行打开详情弹窗，展示基础信息、会话信息、成本、性能和元数据 JSON。
- 上游 `row_to_request_log_detail` helper 已合并本地扩展字段，修复详情查询字段映射不完整的问题。

### 3.7 添加供应商查重

- 添加供应商时按“应用 + API 请求地址 + API Key”查重。
- 请求地址查重前会去掉末尾 `/`，并兼容带 `/v1` 与不带 `/v1` 的输入。
- 只在请求地址和 API Key 均可识别时拦截，避免误伤 OAuth、官方或尚未填写完整的供应商。
- 已增加前端回归测试，覆盖 `/v1/` 与无 `/v1` 的重复识别。

### 3.8 编辑保存细节

- Gemini 编辑提交前会再次把输入框中的 API Key、Base URL、Model 覆盖回 `settingsConfig.env`，避免最后一次输入被状态同步时序丢失。
- 编辑供应商保存时 `pinToTop` 默认值改为 `false`；只有用户主动勾选“置顶”才调整顺序。
- 添加/编辑弹窗的保存选项与提交 payload 解耦，保证保存成功但置顶/启用应用失败时给出独立错误提示。
- Hermes 编辑供应商时，`providerKey` 变更现在会正确映射到新的 provider ID，和 OpenCode / OpenClaw 保持一致。
- 切换模式应用（Claude / Codex / Gemini）在首次添加供应商时，如果用户取消勾选“启用”，后端不再自动把该供应商设为当前并写入 live。
- `src/lib/api/config.ts` 中 `omo-slim` 的前端类型别名已与实际参数统一，避免类型约束与真实命令值漂移。

### 3.9 供应商列表搜索与实时活动

- 供应商搜索改为定位式搜索，不再过滤列表，避免搜索时误以为故障转移队列只剩命中项。
- 搜索框长期固定在顶部操作栏右侧，支持 Enter / Shift+Enter 上下跳转。
- 搜索命中行高亮，右侧显示结果小地图，适合大量供应商时快速定位。
- 表格增加“顶部 / 上一页 / 下一页 / 底部”滚动按钮。
- 默认窗口尺寸调整为 `1480x900`，最小尺寸调整为 `1260x720`，表格最小宽度为 `1360px`。
- 表格改为固定功能列 + 自适应名称列：操作列按 switch 与图标按钮数量收窄到 `272px`，状态列按多 tag 组合收窄到 `206px`，名称列使用剩余宽度。
- 主界面增加实时活动条，基于代理内存态 `active_request_targets` 展示当前正在处理的应用、供应商、模型和并发数量，不进入数据库。
- 状态列继续显示单供应商处理状态；全局活动条用于多应用、多供应商并发场景的快速判断。

## 4. 配置处理最终规则

所有应用使用同一条决策链路：

1. 供应商数据库配置是原始事实源。
2. 根据 `meta.useConfigTemplate` 判断是否套用模板。
3. 模板替换 `{providerConfig}` / `{mcpConfig}` / `{settingsConfig}`。
4. 交给应用对应 live 写入适配器。
5. 代理接管、故障转移、普通切换都使用同一有效配置生成函数。

特殊说明：

- Codex 的 `{mcpConfig}` 会在存在 MCP 时生成完整 `[mcp_servers]`，不存在时为空。
- Gemini 的 `.env` 和 `settings.json` 分别渲染，`mcpServers` 会统一覆盖为当前启用集合，避免旧值残留。
- OpenCode / OpenClaw / Hermes 属于累加模式，模板作用于单供应商片段，不强制覆盖完整应用配置文件。

## 6. 上游对比原则

- 不直接 merge 上游分支，避免覆盖本地定制。
- 对上游提交按功能逐项移植，并在冲突处保留本地定制语义。
- 对涉及核心流程的上游改动，按“上游修复意图 + 本地扩展字段”重新合并，例如用量统计 `RequestLogDetail` 同时保留上游 helper 和本地会话标题/项目路径字段。
