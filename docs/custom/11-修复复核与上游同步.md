# 修复复核与上游同步记录

## 1. 当前基线

本地仓库：`https://github.com/ppengit/cc-switch`

上游仓库：`https://github.com/farion1231/cc-switch`

本地定制目标：在持续接收上游修复的前提下，保证供应商管理、代理接管、故障转移、MCP、请求日志和配置文件处理保持稳定一致。

## 2. 本轮配置体系定稿

最终配置体系为：

- 供应商配置是唯一真实供应商事实源。
- 应用接入配置模板只负责代理接管 live 文件。
- MCP 配置由 MCP 管理数据统一决定。
- 普通直连写真实供应商配置。
- 代理接管写本地代理配置。
- 故障转移由代理运行时切换上游，不依赖改写本地 CLI live 文件。

配置体系不再提供独立的供应商默认配置模板功能，也不再在添加/编辑供应商时提供“是否套模板”的保存选项。

## 3. 默认配置更新

根据 2026-04-30 官方资料和当前编码代理使用场景更新默认值：

- Codex / OpenAI：`gpt-5.5`，Responses API，`model_reasoning_effort="high"`。
- Claude：`claude-sonnet-4-6`，`claude-haiku-4-5-20251001`，`claude-opus-4-7`。
- Gemini：`gemini-3.1-pro-preview`；Flash 预设使用 `gemini-3-flash-preview`。
- OpenCode / OpenClaw / Hermes：默认 starter 预置 OpenAI Responses / Codex Responses 结构和 `gpt-5.5`，聚合供应商预设保留各自协议。
- TheRouter 等聚合平台的默认 Codex 模型改为 `openai/gpt-5.5`，旧 Codex 专用模型仅作为可选模型保留。
- Claude / Codex / Gemini 官方预设不再是空配置：保留官方认证方式，同时预置模型和必要的低噪声/隐私默认项。
- 后端默认应用接入模板已收敛到 `src-tauri/src/app_config_templates.rs`，配置弹窗和代理接管共享同一份默认模板，避免以后更新模型或模板时多处漂移。
- OpenCode starter 补齐 `small_model` 与 `setCacheKey`；OpenClaw starter 补齐上下文窗口和最大输出 token。

## 4. 已修复的核心问题

### 4.1 Gemini Key 保存丢失

- 编辑供应商只以数据库 provider 为表单事实源。
- Gemini API Key、Base URL、Model 输入会同步回 `settingsConfig.env`。
- 提交前再次以输入框状态覆盖 JSON，避免最后一次输入被 React 状态时序吞掉。

### 4.2 置顶顺序重复

- 置顶后后端归一化同应用所有供应商排序。
- 表格序号按展示队列计算，不直接暴露原始 `sort_index`。

### 4.3 熔断、降级、故障转移状态

- 状态列重新接入真实代理健康状态和熔断器统计。
- 当前代理、处理中、熔断、半开、降级均来自运行时状态。
- `get_circuit_breaker_stats` 从运行中的代理路由读取，不再返回空数据。

### 4.4 接管期间真实配置泄露风险

- 接管 live 始终由应用接入配置模板生成。
- live 文件只写本地代理地址和 `PROXY_MANAGED`。
- 热切换、模板保存、MCP 变更都会先更新 live backup，再重建接管 live。

### 4.5 Claude 接管模型字段被错误清理

- 接管 live 中的 Claude 默认模型现在由应用接入配置模板控制。
- 热切换不再删除模板生成的 `ANTHROPIC_MODEL`、Haiku/Sonnet/Opus 默认模型映射。
- 这样既避免旧供应商模型残留，也避免接管模板默认值被误删。

### 4.6 请求实时活动回落

- 代理内存态记录正在处理的应用、供应商、模型和并发数。
- SSE / 流式响应增加 drop guard，客户端断开或流提前丢弃也会释放活动状态。
- 顶部活动条和供应商状态列使用同一份运行时状态，无请求时及时回落。

### 4.7 请求日志详情

- 请求日志列表补充会话标题和项目路径。
- 双击请求日志可查看元数据详情。
- 详情查询字段与列表字段保持一致。

### 4.8 实时请求模型显示错误

- 代理活动状态不再只看请求初始模型。
- 请求进入代理后，会按“请求上下文模型 -> 实际出站请求体模型 -> 流式事件模型 -> 最终响应模型”逐步刷新。
- Claude 转换流、Responses SSE、Gemini 转换流都会把解析到的真实模型回写到活动状态。
- 同一请求再次刷新模型时只更新模型与时间戳，不重复增加并发计数。

### 4.9 顶部活动条单请求多 Tag

- 前端增加活动目标归一化层，对 `active_request_targets` 先按 `app_type + provider_id` 去重。
- 若后端瞬时快照出现“`active_request_count = 1` 但目标列表大于 1”的不一致状态，活动条和代理面板只展示最新目标。
- 当前应用供应商列表中的“处理中”来源也改为使用同一套归一化后的活动目标，避免顶部和列表状态不一致。
- 补充了一条真实代理集成测试：客户端请求模型为 `gpt-5.4`，上游流式响应模型为 `gpt-5.3-codex`，验证请求处理中状态会刷新到真实上游模型。

### 4.10 编辑供应商输入回退

- 编辑供应商弹窗打开后冻结当前供应商快照，只在重新打开或切换供应商时刷新表单初始值。
- `ProviderForm` 的 `reset` 改为按 `appId + providerId + mode` 稳定 key 触发，避免供应商列表状态刷新、代理活动刷新或父组件重渲染把正在输入的内容回填成旧值。
- 增加真实表单回归测试，覆盖“同一供应商 ID 父级重渲染后用户输入仍保留”。

### 4.11 请求日志详情层级

- 请求日志双击详情弹窗提升到 `top` z-index 层级。
- 详情弹窗仍走 `DialogPortal`，但不再被全屏面板、配置弹窗或其它顶层内容遮挡。

### 4.12 顶部实时活动条高度

- 顶部实时活动条改为固定高度占位。
- 有无请求时只切换内部内容透明度，不再新增/移除整块 DOM 高度，避免供应商列表页面上下抖动。

### 4.13 供应商配置模板编辑

- 供应商配置模板内部仍按原 JSON 结构保存，避免数据库结构变更。
- UI 按应用拆分配置文件显示：Codex 展示 `auth.json` 与 `config.toml`，Gemini 展示 `.env` 与 `settings.json`。
- 默认模板和加载后的模板会格式化展示，不再把 `auth`、`env`、`config.toml` 混在一个难读 JSON 字符串里。

### 4.14 请求日志详情回退显示

- 请求日志双击后，详情弹窗优先使用列表行里已有的摘要数据立即展示，不再因为详情接口尚未返回就先落到“请求未找到”。
- 若详情接口成功，弹窗自动升级为完整记录；若详情接口失败，则保留摘要展示并明确提示“当前展示的是列表摘要信息”。
- 详情弹窗最大宽高同步放大，统一走应用根部 `DialogPortal` 顶层渲染，避免继续被设置页或全屏面板裁切。

### 4.15 代理原始日志面板

- 在请求日志、Provider 统计、模型统计后新增“代理原始日志”页签。
- 数据源改为代理运行期内存环形缓冲区，只保留链路元数据，不落库、不保存请求体响应体，也不记录 API Key。
- 当前记录的事件包括：
  - `received`：代理已收到请求。
  - `routed`：已选定供应商并开始向上游发送。
  - `finished`：请求完成。
  - `failed`：请求失败。
- 双击单条日志可看原始结构化详情，便于排查“应用是否真的打到代理、路由到了谁、最终成功还是失败”。

### 4.16 供应商状态提示增强

- 状态列继续保留“当前代理 / 处理中 / 熔断 / 半开 / 降级”等标签，但增加颜色区分：
  - 当前代理、处理中：绿色。
  - 启用：蓝色。
  - 熔断：红色。
  - 半开：橙色。
  - 降级：黄色。
- 状态标签增加 tooltip，展示连续失败次数、熔断统计、最后错误、最后失败时间，以及正在处理的真实模型。
- 这样可以把“状态是什么”和“为什么是这个状态”同时暴露出来，减少误判。

## 5. UI 与交互修复

- 供应商列表改为紧凑表格。
- 搜索改为定位，不筛选数据。
- 搜索框下方弹出命中供应商列表，可点击定位。
- 增加顶部、上一页、下一页、底部快速滚动按钮。
- 启用/禁用改为 switch，放到名称列左侧。
- 去掉供应商 Logo 列占位。
- 置顶按钮统一放在编辑按钮后。
- Claude 的打开终端按钮放到操作列最后。
- 配置相关弹窗增大并补齐关闭按钮和边距。
- 主窗口默认尺寸调整为 `1680x1040`，最小尺寸为 `1380x820`。

## 6. 数据导出策略

供应商迁移脚本只导出 `providers` 表，避免污染新库：

- 供应商名称、官网、备注、图标、分类。
- API Key、API 请求地址、模型和协议参数等 `settings_config` 内容。
- 启用状态、当前供应商、故障转移队列、排序、价格倍率和限额。

明确不导出：

- `settings` 表中的旧通用配置、供应商默认模板、应用接入配置模板。
- MCP 服务器配置。

不导出会话、请求日志、用量统计、窗口设置、WebDAV、技能等非供应商域数据。

生成物为 `release/provider-migration/restore_providers_*.py`，脚本自包含数据，不再依赖额外 JSON bundle。

## 8. 上游同步原则

- 不直接覆盖本地定制。
- 上游修复按功能逐项移植。
- 涉及代理、接管、故障转移、日志、用量统计的上游改动必须重新走本地扩展字段和定制流程复核。
