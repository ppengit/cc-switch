# 配置模板与当前配置编辑方案（最终版）

## 1. 配置边界

最终版本只保留三类配置对象：

- 供应商配置：保存在 `providers.settings_config`，是 API Key、API 请求地址、模型、协议参数等真实供应商信息的唯一事实源。
- MCP 配置：保存在 MCP 管理数据中，由应用适配器按各应用格式写入或同步。
- 应用接入配置模板：只用于代理接管时生成目标应用的 live 配置，使应用连接 cc-switch 本地代理，而不是直接连接真实供应商。

不再提供额外的供应商默认模板功能，也不再让单个供应商选择是否套用模板。供应商编辑框里看到和保存的就是该供应商自身配置。

## 2. 应用接入配置模板

应用接入配置模板按应用、按文件保存，存储键为 `config_template_{app}`，值为 `AppConfigTemplateFile[]` 的 JSON 字符串。

后端默认模板只维护在 `src-tauri/src/app_config_templates.rs`：

- 配置弹窗读取默认模板时使用该模块。
- 代理接管重建 live 配置时使用该模块。
- 如果数据库中保存的是旧占位符模板，会自动回退到该模块的默认模板。
- `src/components/providers/ProviderList.tsx` 只保留前端接口失败时的兜底展示，内容需与后端默认模板保持一致。

每个模板文件包含：

- `key`：稳定文件标识，例如 `auth`、`config`、`env`、`settings`。
- `label`：界面显示名，例如 `auth.json`、`config.toml`、`.env`、`settings.json`。
- `content`：模板文本。

Claude / Codex / Gemini 的应用接入模板支持代理接管所需占位符：

- `{proxyBaseUrl}`：cc-switch 本地代理根地址。
- `{proxyCodexBaseUrl}`：cc-switch 本地代理 OpenAI/Codex `/v1` 地址。
- `{proxyToken}`：接管态占位 Token，默认 `PROXY_MANAGED`。
- `{mcpConfig}`：当前应用启用的 MCP 配置片段；无启用 MCP 时为空，并移除对应空行。

默认模板：

- Claude：`settings.json`，写入 `ANTHROPIC_BASE_URL`、`ANTHROPIC_AUTH_TOKEN`、Sonnet 4.6、Haiku 4.5、Opus 4.7 默认模型映射和 `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1`。
- Codex：`auth.json` + `config.toml`，默认 `gpt-5.5`、Responses API、`model_reasoning_effort="high"`、本地代理 `/v1`。
- Gemini：`.env` + `settings.json`，默认 `gemini-3.1-pro-preview`、API Key 认证模式和按需 `mcpServers`。
- OpenCode：当前不支持代理接管，默认模板提供可直接编辑的 `opencode.json` starter：`@ai-sdk/openai`、`https://api.openai.com/v1`、`gpt-5.5`、`small_model`、`setCacheKey=true`、空 `mcp`。
- OpenClaw：当前不支持代理接管，默认模板提供可直接编辑的 JSON5 starter：`openai-responses`、`https://api.openai.com/v1`、`gpt-5.5`、`contextWindow=400000`、`maxTokens=128000`。
- Hermes：当前不支持代理接管，默认模板提供可直接编辑的 YAML starter：`codex_responses`、`https://api.openai.com/v1`、`gpt-5.5`。

## 3. 普通直连模式

未开启代理接管时，应用实际配置由“当前供应商配置 + 当前启用 MCP”生成：

- Claude：供应商 `settings_config` 写入 Claude settings；MCP 仍按 Claude 独立机制同步，不混写到 settings。
- Codex：供应商 `auth` 与 `config.toml` 写入 live，并用 MCP 管理中的启用集合重建 `[mcp_servers]`。
- Gemini：供应商 `.env` 与 `settings.json` 写入 live，并用 MCP 管理中的启用集合重建 `mcpServers`；无启用 MCP 时移除 `mcpServers`。
- OpenCode / OpenClaw / Hermes：累加模式，同一 live 配置内可存在多个供应商；启用/禁用供应商会增删对应 provider 项。

普通直连模式不使用应用接入配置模板，因为此时应用应直接读取供应商自己的真实配置。

## 4. 代理接管模式

开启代理接管后，目标应用 live 配置永远写成稳定的本地代理接入配置：

- live 中只保留 cc-switch 本地代理地址。
- live 中只保留 `PROXY_MANAGED` 占位 Token。
- live 中不得泄露真实供应商 API Key 或真实供应商地址。
- 供应商真实配置保存在数据库和 live backup 中，由代理转发请求时读取。

切换供应商、保存应用接入模板、MCP 变更、故障转移都会走同一条路径：

1. 从数据库读取当前目标供应商配置。
2. 合并当前应用启用的 MCP 集合，生成可恢复的真实供应商配置。
3. 更新 live backup，保证关闭接管时恢复到最新供应商和最新 MCP。
4. 用应用接入配置模板重建 live，使应用继续连接本地代理。

## 5. 故障转移时的配置行为

故障转移必须依赖代理接管才能保证稳定，因为用户可能从外部终端直接启动应用，而不是从 cc-switch 打开终端。

当开启代理接管和故障转移时：

- 应用 live 配置不会随着 A 供应商切到 B 供应商而写成 B 的真实配置。
- 本地应用始终连接 cc-switch 代理。
- 代理运行时根据熔断、降级、手动切换或故障转移策略选择真实供应商。
- 真实上游请求使用当前被选中供应商的 API Key、base_url、协议参数和模型映射。

因此如果 A 供应商有参数 `a=1`，B 供应商有参数 `a=2`：

- 该参数属于上游请求或代理转发参数时，A 熔断切到 B 后，代理会使用 B 的 `a=2`。
- 该参数属于本地 CLI 启动行为时，切到 B 不会自动改变本地 CLI live 配置；这类参数应放到应用接入配置模板或应用级配置中，而不是做成供应商私有参数。

## 6. 当前配置编辑

“当前配置”展示应用实际 live 文件内容：

- Gemini 会同时展开 `.env` 与 `settings.json` 两个文本框。
- Codex 会同时展开 `auth.json` 与 `config.toml`。
- Claude 展开 `settings.json`。
- 所有文本框直接展示实际文件内容，不使用下拉选择。

保存规则：

- 未接管时，可以直接保存到应用 live 文件，但不会自动反向写回供应商配置。
- 已接管时，Claude / Codex / Gemini 禁用直接保存，避免绕过接管链路写入真实地址或真实 Key。
- 可通过“回显到 MCP 管理”把 live 中的 MCP 配置导回 MCP 管理数据。

## 7. 默认配置目标

新装后应尽量可用，而不是从空配置开始：

- 新建 Codex 供应商默认包含 `auth.OPENAI_API_KEY` 和 `config.toml` 基础结构，只需填写 Key 和 base_url。
- 新建 Gemini 供应商默认包含 `.env` 的 Key/base_url/model 与 `settings.json` 的认证结构。
- 新建 Claude 供应商默认包含 API Key/base_url/model 映射结构。
- OpenCode / OpenClaw / Hermes 默认包含 GPT-5.5 或对应 OpenAI-compatible 结构，减少手写 JSON/YAML 成本。
- Claude / Codex / Gemini 的官方预设也带模型默认值；官方认证仍由各 CLI 自身处理，不强制写空 Key。
