# 供应商列表与交互设计

## 1. 列表改造目标

把供应商卡片列表改为紧凑表格，同时保留拖拽排序和故障转移顺序的可控性。

## 2. 表格列定义

- 选择：checkbox，表头支持全选/反选。
- 序号：按故障转移顺序显示。
- 供应商名称：名称本身作为链接载体；无官网链接时从请求地址推导，链接展示会裁剪 `/v1` 或 `/v1/...`。
- 备注。
- 模型名称：按应用类型提取默认模型或模型集合摘要。
- 状态：支持点击列头排序，但只影响前端展示，不改故障转移顺序。
- 操作：启用/禁用、编辑、置顶、复制、测试、删除、用量配置、打开终端，并保留原有条件判断。

## 3. 顶部操作栏

- 全部启用/禁用：未选择时作用于全部，选择后只作用于选择项。
- 状态统计：`N/M`，N 为启用数量，M 为总数量。
- 删除：未选中禁用，选中后显示数量。
- 最近会话：按当前应用展示最近 10 条，可直接调起恢复命令到终端。
- 应用接入配置模板：编辑代理接管时写入目标应用 live 的模板。
- 当前配置：直接查看和编辑应用实际配置文件。
- 搜索：长期显示在操作栏右侧，执行定位而不是过滤。

## 4. 拖拽与排序规则

- 行拖拽保持可用。
- 拖拽后走后端排序更新，并归一化同应用排序值。
- 状态列排序与拖拽顺序独立，不反写数据库排序。
- 搜索不隐藏行，避免误判故障转移队列。

## 5. 当前代理状态显示

状态列显示：

- 当前代理。
- 启用/禁用。
- 处理中及并发数量。
- 熔断。
- 半开。
- 降级及连续失败次数。

状态数据优先来自代理运行时状态，避免 UI 自行推断。

## 6. 添加/编辑保存选项

保存选项外置到“添加/编辑供应商”底部按钮栏左侧：

- 置顶：保存后顺序置为第一，并同步故障转移顺序。
- 启用：保存后立即生效；故障转移模式进入队列，累加模式写入对应应用 live provider。

默认值：

- 添加供应商：`置顶=true`、`启用=true`。
- 编辑供应商：`置顶=false`、`启用=当前供应商启用状态`。

## 7. 搜索定位与快速滚动

供应商搜索从“过滤列表”改为“定位结果”：

- 搜索范围：供应商名称、备注、官网链接、请求地址、模型显示值。
- 命中行高亮，当前定位行使用更明显边框。
- 支持 Enter / Shift+Enter 在结果间跳转。
- 搜索框下方显示结果弹层，每项展示供应商名称、序号、启用状态、当前代理、处理中等 tag。
- 表格顶部提供“顶部 / 上一页 / 下一页 / 底部”滚动按钮。

布局调整：

- 主窗口默认尺寸 `1680x1040`，最小尺寸 `1380x820`。
- 表格最小宽度 `1340px`。
- 操作列和状态列按实际按钮/tag 宽度收敛，名称列自适应承载主要文本。
- 列表和配置编辑文本域字号放大，降低高分屏阅读成本。
- “切换到此供应商 / 使用此供应商”按钮改用独立的右向切换图标，不再与“置顶（顺序 1）”共用上箭头图标，避免两个动作的视觉语义冲突。

## 8. 全局实时活动条

主界面收到代理请求并仍在处理时显示实时活动条：

- 数据来源为代理内存态，不写入数据库。
- 按应用 + 供应商分组展示。
- 支持多个应用、多个供应商并发。
- 每个 chip 展示应用、供应商、模型和并发数量。
- 请求正常结束、失败、超时、客户端断开时都会及时回落。
- 模型字段优先取真实出站请求体中的 `model`，后续若流式事件或最终响应暴露了实际响应模型，会继续覆盖刷新，避免把默认模型误显示成实际处理模型。
- 展示前会先按 `app_type + provider_id` 去重归一化；如果后端瞬时上报出现“1 个请求但多个目标 tag”的不一致状态，前端会优先保留最新目标，避免活动条误显示多个 chip。
