# 各应用官方配置文档调研（2026-04-30）

## 1. 调研目标

目标是把新安装后的默认配置做成可直接编辑使用的起点，同时保证配置文件写法贴合各应用官方格式。

本次只采用官方或项目主页资料作为依据，具体链接：

- OpenAI 最新模型说明：https://developers.openai.com/api/docs/guides/latest-model
- OpenAI Codex 配置参考：https://developers.openai.com/codex/config-reference
- Anthropic Claude 模型列表：https://docs.anthropic.com/en/docs/about-claude/models/overview
- Claude Code 设置：https://docs.anthropic.com/en/docs/claude-code/settings
- Gemini 模型列表：https://ai.google.dev/gemini-api/docs/models/gemini-3.1-pro-preview
- Gemini CLI 配置：https://github.com/google-gemini/gemini-cli/blob/main/docs/cli/configuration.md
- OpenCode 配置：https://opencode.ai/docs/config
- OpenCode Providers：https://opencode.ai/docs/providers
- OpenCode MCP：https://opencode.ai/docs/mcp
- OpenClaw 配置：https://docs.openclaw.dev/getting-started/configuration
- Hermes Agent 配置：https://github.com/NousResearch/hermes-agent/blob/main/website/docs/user-guide/configuration.md

## 2. 默认模型选择

默认模型按“当前官方推荐 + 编码/代理场景可用性”选择：

- OpenAI / Codex：`gpt-5.5`。
- Codex reasoning：默认 `high`，适配编码代理的复杂任务；用户仍可在配置中改成 `medium` 或 `xhigh`。
- Claude：默认 `claude-sonnet-4-6`，快速模型 `claude-haiku-4-5-20251001`，高阶模型 `claude-opus-4-7`。
- Gemini：默认 `gemini-3.1-pro-preview`；Flash 场景使用 `gemini-3-flash-preview`。
- OpenCode / OpenClaw / Hermes：优先给 OpenAI-compatible / Responses 结构预置 `gpt-5.5`，同时保留 Claude、Gemini、Qwen、Kimi 等聚合平台常用模型。

代码落点：

- 后端应用接入模板单一事实源：`src-tauri/src/app_config_templates.rs`。
- 前端供应商预设：`src/config/*ProviderPresets.ts`。
- 新建供应商默认配置：`src/components/providers/forms/helpers/opencodeFormUtils.ts`。

## 3. Claude Code

官方要点：

- 用户配置主要在 Claude Code settings 中维护。
- 环境变量可控制 API 地址、Token、模型和默认模型映射。
- MCP 不应混进 Claude settings 主配置，由 Claude 自身 MCP 文件/机制处理。

本项目默认：

- `ANTHROPIC_BASE_URL`：供应商 API 请求地址。
- `ANTHROPIC_AUTH_TOKEN` 或 `ANTHROPIC_API_KEY`：供应商 Key。
- `ANTHROPIC_MODEL`：`claude-sonnet-4-6`。
- `ANTHROPIC_DEFAULT_HAIKU_MODEL`：`claude-haiku-4-5-20251001`。
- `ANTHROPIC_DEFAULT_SONNET_MODEL`：`claude-sonnet-4-6`。
- `ANTHROPIC_DEFAULT_OPUS_MODEL`：`claude-opus-4-7`。
- `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1`：减少非必要网络流量。

## 4. Codex

官方要点：

- 主配置文件为 `~/.codex/config.toml`。
- 支持 `model`、`model_provider`、`[model_providers.<id>]`。
- MCP 使用 `[mcp_servers.<id>]` 子表。
- OpenAI-compatible / Responses 供应商应显式配置 `wire_api="responses"` 和 `base_url`。

本项目默认：

```toml
model_provider = "custom"
model = "gpt-5.5"
model_reasoning_effort = "high"
disable_response_storage = true

[model_providers.custom]
name = "custom"
wire_api = "responses"
requires_openai_auth = true
base_url = "https://api.example.com/v1"
```

`auth.json` 默认包含 `OPENAI_API_KEY`，避免用户需要手写 JSON。

## 5. Gemini CLI

官方要点：

- Gemini API 当前复杂推理/编码默认优先 `gemini-3.1-pro-preview`。
- Gemini CLI 使用 `.env` 承载 API Key、base_url、模型等环境变量。
- `settings.json` 可承载 `mcpServers` 和认证相关设置。

本项目默认：

```env
GOOGLE_GEMINI_BASE_URL=https://api.example.com
GEMINI_API_KEY=
GEMINI_MODEL=gemini-3.1-pro-preview
```

```json
{
  "model": {
    "name": "gemini-3.1-pro-preview"
  },
  "security": {
    "auth": {
      "selectedType": "gemini-api-key"
    }
  }
}
```

MCP 启用时再写入 `mcpServers`，未启用时不写空对象。

## 6. OpenCode

官方要点：

- 主配置支持 provider、model、mcp 等结构。
- Provider 通常依赖 AI SDK 包名，例如 `@ai-sdk/openai`、`@ai-sdk/openai-compatible`、`@ai-sdk/anthropic`。
- OpenAI Responses / GPT-5.5 starter 优先使用 `@ai-sdk/openai`，聚合兼容站点继续使用 `@ai-sdk/openai-compatible`。

本项目默认：

- 应用 starter 使用 `@ai-sdk/openai`、`https://api.openai.com/v1`、`{env:OPENAI_API_KEY}` 和 `gpt-5.5`。
- starter 同时设置 `small_model="openai/gpt-5.5"`；OpenCode 官方文档说明 `small_model` 用于标题生成等轻量任务，未配置时会尝试自动选择便宜模型。
- provider options 默认包含 `setCacheKey=true`，贴合 OpenCode 官方 provider options。
- 自定义聚合供应商仍可使用 `@ai-sdk/openai-compatible`。
- 默认 `options.baseURL`、`options.apiKey`、`models` 均有结构占位。
- 预设中优先包含 `gpt-5.5`，并按供应商能力补充 Claude/Gemini/国产模型。

## 7. OpenClaw

官方要点：

- 配置文件为 OpenClaw JSON 配置。
- Provider 需要 baseUrl、apiKey、api 模式和 models。

本项目默认：

- 自定义供应商默认 `api="openai-responses"`。
- 应用 starter 使用 `https://api.openai.com/v1`、空 API Key 占位、`gpt-5.5`、`contextWindow=400000`、`maxTokens=128000`。
- 聚合平台预设保留 `openai-completions` 或供应商实际需要的 API 模式。

## 8. Hermes Agent

官方要点：

- Hermes 使用 YAML 配置，包含 provider、model、mcp_servers 等结构。
- provider 字段应保留扩展参数，避免写入时丢失用户手动配置。

本项目默认：

- 自定义供应商默认 `api_mode="codex_responses"`。
- 应用 starter 使用 `https://api.openai.com/v1`、空 API Key 占位和 `gpt-5.5`。
- 聚合平台预设保留供应商原生模型命名，例如 `anthropic/claude-sonnet-4.6`、`google/gemini-3.1-pro-preview`。
