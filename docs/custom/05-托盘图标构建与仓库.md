# 托盘、图标、构建与仓库策略

## 1. 托盘交互

- 左键双击：打开主界面。
- 右键：打开菜单。
- 左右键行为不再等同。

## 2. 图标替换

图标来源：`D:\Solution\cc-switch\assets\zelda`

替换范围：

- 托盘图标
- 程序窗口图标
- 打包图标（桌面/任务栏/安装产物）

实施方式：

- 使用 `pnpm tauri icon assets/zelda/logo-mark-1024.png -o src-tauri/icons` 统一重建多平台图标。
- macOS 托盘模板图标同步替换为 Zelda 单色资源（`src-tauri/icons/tray/macos/*`）。

## 3. 构建输出目录

将构建产物统一输出到：

- `根目录/release`

实现方式：

- 新增 `src-tauri/.cargo/config.toml`
- 设置 `[build] target-dir = "../release"`

## 4. 仓库关系

- `origin` 指向：`https://github.com/ppengit/cc-switch`
- 新增 `upstream`：`https://github.com/farion1231/cc-switch`

用于：

- 个人仓库直接发布。
- 后续从 upstream 拉取更新并合并。

应用内链接与更新地址同步切换到 `ppengit`：

- 主界面仓库链接
- About 页面 Release 链接
- Tauri Updater endpoint
- Rust package `repository` 字段
