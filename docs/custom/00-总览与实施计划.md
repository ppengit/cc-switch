# CC Switch 个性化定制实施总览

## 1. 目标

本次定制基于上游最新代码，重点提升以下能力：

- 供应商列表的可读性、可批量操作性、可控性。
- 会话恢复与会话管理效率。
- 配置模板/当前配置的可编辑性与稳定写入。
- 请求日志可追踪性（会话标题、项目路径、元数据查看）。
- 托盘交互与全量图标统一。
- 后续可持续对接上游仓库更新。

## 2. 实施范围

本次实施覆盖以下模块：

- 前端：供应商列表、添加/编辑供应商、会话管理、请求日志。
- 后端：会话命令、配置命令、使用统计查询、托盘行为。
- 数据库：会话标题映射与配置模板相关存储。
- 工程配置：构建输出目录、图标资源、Git 远程仓库配置。
- 文档调研：各应用官方配置文档映射（见 `07-各应用官方配置文档调研.md`）。

## 3. 设计原则

- 排序语义分离：
  - 故障转移顺序只由拖拽/置顶维护。
  - 状态列排序只影响当前页面展示，不反写顺序。
- 写配置可回溯：
  - 模板+占位符替换采用锚点/分段替换，避免整段覆盖导致混乱。
- 兼容优先：
  - 尽量不破坏现有命令与数据结构，新增字段与接口向后兼容。
- 可扩展：
  - 会话标题映射仅本地数据库，不回写应用原始会话文件。

## 4. 里程碑

1. 文档落地（本目录）。
2. 后端能力补齐（会话、配置、日志、托盘）。
3. 供应商列表表格化重构与批量操作。
4. 表单增强与请求日志增强。
5. 图标/构建/远程仓库收尾与回归验证。

## 5. 可用性增强建议（超出原始需求）

- 增加“供应商变更历史”面板（近 50 条启用/禁用/置顶/删除记录）。
- 增加“排序锁定”开关（防误拖拽）。
- 提供“配置差异预览”（模板替换前后 diff）。
- 请求日志支持“按会话聚合”视图。
- 会话恢复支持“多终端预览命令”与“一键复制+启动”。

## 6. 2026-04-30 修复复核补充

本轮在上游 `eb304232` 基础上继续整改，重点修复：

- 配置模板改为多文件结构，Gemini `.env` 与 `settings.json` 分开编辑和保存。
- 配置模板保存支持立即同步到实际环境配置。
- 当前配置弹窗直接展开全部配置文件，不再使用单文件下拉选择。
- 旧“通用配置”不再作为主流程，供应商原始配置与模板渲染结果分离。
- Gemini 编辑保存以数据库供应商为单一事实源，不再用 live 渲染结果覆盖，避免 API Key 丢失。
- 供应商排序保存会重新归一化同应用所有 `sort_index`，避免置顶后出现重复序号。
- 熔断、降级、当前代理状态重新接入真实运行中的 `ProviderRouter` 统计。
- 已移植上游新增用量去重、Hermes Agent 用量、SiliconFlow USD、Kimi URL、Codex 1M 上下文隐藏、README sponsor 等提交。
