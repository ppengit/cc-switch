# CC Switch v3.12.2+2

> 上流 v3.12.2 を土台にした完全統合版。アプリ設定テンプレート、最終設定プレビュー、セッションルーティング、モデル提案フローを全面的に整理

## 概要

CC Switch v3.12.2+2 は、上流 v3.12.2 をベースにローカルのカスタマイズ機能を維持したまま完成させた統合リリースです。今回の主な焦点は、アプリ設定テンプレートとデフォルトプロバイダーテンプレートの責務分離、最終設定プレビューの完全レンダリング、カスタム版セッション管理とセッションルーティング体験の復元、そして Claude / Codex / Gemini のデフォルトモデル更新です。

**リリース日**: 2026-03-22

**更新規模**: 57 commits | 256 files changed | +23,386 / -8,046 lines

---

## ハイライト

- **アプリ設定テンプレート / デフォルトプロバイダーテンプレートの分離**: Claude、Codex、Gemini でテンプレート責務を明確化し、設定汚染を減少
- **最終設定プレビューの強化**: 完全レンダリング、差分表示、live ファイルの直接編集・保存に対応
- **Codex 設定レンダリングの安定化**: `{{provider.config}}` と任意の `{{mcp.config}}` を使った完全テンプレートレンダリングに対応し、テイクオーバー時はローカルプロキシアドレスを正しく表示
- **セッション管理 / ルーティング整理**: カスタム版の会話記録中心の構成を復元し、ラベルやデフォルト挙動を整理
- **モデル提案サポート**: OpenAI 互換 `/models` API を持つエンドポイント向けにモデル取得とサジェスト UI を追加

## 変更点

### 設定システム

- 旧来の「共通設定」を **アプリ設定テンプレート** として整理
- **デフォルトプロバイダーテンプレート** は新規 provider 作成時の初期断片として維持
- Codex のデフォルトプロバイダーテンプレートは純粋な TOML のまま保持し、auth 情報を含めないように修正
- 旧式または不正な Codex テンプレートは有効なテンプレート形式へ自動正規化

### 最終設定プレビュー

- 完全にレンダリングされた設定全体を表示するよう改善
- live ファイルの直接編集・保存に対応
- 差分表示を整理し、期待値と live の比較が容易に
- プロキシテイクオーバー有効時、期待値の URL が元の上流ではなくローカルプロキシアドレスを指すよう修正

### セッションとルーティング

- 会話記録リスト中心のカスタム版セッション管理構造を復元
- セッションルーティング関連の文言、デフォルト provider 説明、状態表示を整理
- provider リスト上の関連ステータスとヒントを一貫化

### デフォルトモデルとモデル提案

- テンプレートおよび統合 provider フローの Claude / Codex / Gemini デフォルトモデルを更新
- OpenAI 互換 endpoint に対するモデル一覧取得を追加
- モデル入力欄に共通サジェスト UI を導入

### バッチヘルスチェック

- テスト用プロンプトを複数行で設定可能に変更
- provider ごとのチェック実行時に、設定済み行から 1 行をランダム選択して使用

---

## 修正

- アプリ設定テンプレートが provider 設定断片へ誤って書き戻される問題を修正
- Codex 最終プレビューでのテンプレートプレースホルダーレンダリング不整合を修正
- provider 操作ボタンの見た目・幅・挙動の不一致を修正
- ダークモードでの provider 検索 hover 表示不具合を修正
- 統合作業中に入り込んだ複数の設定プレビュー／保存／UI の境界不整合を修正
