# CC Switch v3.12.2+2

> 基于上游 v3.12.2 的完整整合发布，重点收口应用配置模板、最终配置预览、会话路由与模型建议链路

## 概览

CC Switch v3.12.2+2 是基于上游 v3.12.2 完成整合后的正式版本，同时完整保留了本地定制化能力。本次版本重点在于彻底拆分应用级配置模板与供应商默认模板职责、让最终配置预览真实反映完整 live 配置、恢复并收口定制版会话管理与会话路由体验，以及更新 Claude、Codex、Gemini 的默认模型与模型建议链路。

**发布日期**：2026-03-22

**更新规模**：57 commits | 256 files changed | +23,386 / -8,046 lines

---

## 重点内容

- **应用配置模板 / 默认供应商模板彻底分离**：Claude、Codex、Gemini 现在明确区分“应用配置模板”和“默认供应商模板”，减少配置污染和预览偏差
- **最终配置预览升级**：支持完整渲染、实时 diff、高亮对比，并可直接编辑和保存当前 live 文件
- **Codex 配置渲染稳定化**：Codex `config.toml` 现在基于完整应用模板渲染，支持 `{{provider.config}}` 与可选 `{{mcp.config}}`，代理接管时预览会正确显示本地代理地址
- **会话管理 / 会话路由收口**：恢复定制版以对话记录为主体的会话管理结构，并统一相关标签、默认值和说明
- **模型建议支持**：为支持 OpenAI 兼容接口的供应商表单新增自动获取模型与建议下拉

## 变更

### 配置系统

- 将原“通用配置”语义收口为**应用配置模板**
- 将**默认供应商模板**保留为新建 provider 时使用的初始供应商片段
- Codex 默认供应商模板保持纯 TOML，不再混入 auth 内容
- 对遗留的 Codex 非法模板进行自动清洗，统一回退为合法模板格式

### 最终配置预览

- 最终配置预览现在展示完整渲染后的配置，而不再只是供应商片段
- 新增 live 文件直接编辑与保存能力
- 差异展示方式更清晰，便于对照预期结果与当前 live 文件
- 当开启代理接管时，预期结果中的地址会正确显示为本地代理地址，而不是供应商原始请求地址

### 会话与路由

- 恢复以对话记录列表为主体的定制版会话管理结构
- 收口会话路由中的标签、默认提供商说明和相关状态文案
- 统一供应商列表中的路由状态展示与提示

### 默认模型与模型建议

- 更新模板和统一供应商流程中 Claude / Codex / Gemini 的默认模型
- 为支持 OpenAI 兼容接口的场景增加模型列表获取能力
- 模型输入框新增通用建议下拉组件

### 批量健康检查

- 全局测试问题现在支持多行配置
- 每次供应商测试都会随机选择其中一行问题发起检查

---

## 修复

- 修复应用配置模板被误写回供应商配置片段的问题
- 修复 Codex 最终配置预览中模板占位渲染不稳定的问题
- 修复供应商操作区按钮样式、宽度和交互不一致的问题
- 修复深色模式下供应商搜索定位行 hover 样式问题
- 修复整合过程中多处配置预览、落盘与 UI 行为不一致的边界情况
