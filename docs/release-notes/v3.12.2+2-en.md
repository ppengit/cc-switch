# CC Switch v3.12.2+2

> Full v3.12.2 Integration Release with Template Cleanup, Final Config Preview, Session Routing Stabilization, and Model Suggestion Support

## Overview

CC Switch v3.12.2+2 is the finalized integration release built on top of upstream v3.12.2 while preserving the local customization line. This version focuses on fully separating application-level config templates from provider-specific fragments, making final config rendering more reliable, restoring customized session management and session routing behavior, and updating default model presets for Claude, Codex, and Gemini.

**Release Date**: 2026-03-22

**Update Scale**: 57 commits | 256 files changed | +23,386 / -8,046 lines

---

## Highlights

- **Application Template / Provider Template Split**: Claude, Codex, and Gemini now treat app config templates and default provider templates as separate responsibilities, reducing config pollution and preview drift
- **Final Config Preview Upgrade**: Final preview now renders the complete expected live config, supports live-file editing and saving, and shows clearer diff styling
- **Codex Config Rendering Stabilization**: Codex now renders `config.toml` from a full app template with `{{provider.config}}` and optional `{{mcp.config}}`, while takeover preview resolves `base_url` to the local proxy port
- **Session Routing & Session Manager Cleanup**: Restored the customized session manager structure and tightened session routing labels, defaults, and workflow consistency
- **Model Suggestion Support**: Added OpenAI-compatible model fetching and suggestion dropdowns in supported provider forms

---

## Changes

### Config System

- Reframed "common config" into **application config templates**
- Kept **default provider templates** as provider-only initial fragments
- Codex default provider templates now remain pure TOML and no longer mix in auth data
- Legacy invalid Codex provider templates are sanitized back to valid template format

### Final Preview

- Final preview now shows the complete rendered config instead of only partial provider fragments
- Added direct live-file editing and save support inside the preview
- Improved diff presentation to make expected vs. live content easier to inspect
- When proxy takeover is enabled, expected preview output now uses the local proxy address instead of the original upstream URL

### Session & Routing

- Restored the customized session manager layout centered on conversation records
- Cleaned up session routing labels and default-provider wording
- Tightened provider list status display and related routing hints

### Model Defaults & Suggestions

- Updated default Claude/Codex/Gemini models used by templates and universal provider flows
- Added model fetching for supported OpenAI-compatible endpoints
- Added reusable suggestion dropdown support for model fields

### Batch Health Check

- Global test prompts now support multiple lines
- Provider tests randomly pick one configured prompt line per run

---

## Fixed

- Fixed app templates being written back into provider config fragments
- Fixed Codex final preview rendering errors around template placeholders
- Fixed provider action layout issues, including action column width and button consistency
- Fixed dark-mode row hover feedback in provider search results
- Fixed several config and preview edge cases introduced during the integration process
